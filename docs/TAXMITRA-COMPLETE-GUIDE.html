2. The Tools We Use

Each of these is a separate service. Think of them like the contractors building your house — each does one specific job.

2.1 GitHub

What it is: A website that stores your code/files in the cloud. Free.
Why: So your files exist somewhere other than just your laptop. If your laptop dies, nothing is lost.
Account: caswapnilssoni-create
Two repos: fintech-website (marketing) and taxmitra-portal (client portal)
Analogy: Like Google Drive, but specifically for code, with extra features like change history.

2.2 GitHub Desktop

What it is: An app on your computer that talks to GitHub.
Why: It lets you "save" your file changes from your laptop to GitHub with a button click.
Analogy: The courier that delivers your work from your desk to the warehouse.

2.3 GitHub Pages

What it is: Free hosting from GitHub. Turns HTML files into a live website.
Why: Hosts your marketing site. Every push = new version live in ~30 seconds.
Cost: Free forever.

2.4 Vercel

What it is: Hosting service for modern web apps.
Why: Hosts the client portal. The portal needs server features (login, etc.) that Vercel handles.
Cost: Free until thousands of users.
Analogy: GitHub Pages is for a static shop window. Vercel is for a real working office.

2.5 Supabase

What it is: Database + login system + file storage in one.
Region: Mumbai (fast for Indian users).
Cost: Free until ~500 active clients.

What's inside Supabase:

  • Auth — stores signups (emails + passwords)
  • Database — tables for clients, filings, documents, notifications, admins
  • Storage — keeps actual uploaded files (PDFs, JPGs)

Analogy: Filing cabinet + receptionist + secure storage room.

2.6 Resend

Why: When filing status changes, Resend automatically emails the client.
Cost: Free for 3,000 emails/month.
Analogy: A virtual postman.

2.7 Hashnode

Why: You write blogs there in a nice editor. We copy content to your main site for SEO.
Address: blog.taxmitrafinance.com

2.8 Namecheap

2.9 Google Search Console

5. The Blog System

How it works (step by step)

  1. You write a blog post in Hashnode.
  2. In Hashnode, click "..." → Export as Markdown. A .md file downloads.
  3. Edit the first few lines (frontmatter) — title, date, category, etc.
  4. Drop the file into content/blog/ in fintech-website.
  5. Commit + Push via GitHub Desktop.
  6. A GitHub Action runs build-blog.mjs.
  7. The script converts .md → HTML at /blog/<slug>.html.
  8. Updates data/blogs.json (homepage list).
  9. Updates sitemap.xml (so Google knows about the new article).

Total time from writing → live: ~2 minutes.

6. SEO Work

SEO = Search Engine Optimization. How Google decides which sites to show on top.

What we did for your site

1. Sitemap (sitemap.xml) — list of every page given to Google. Auto-updates when you publish a new blog.

2. Robots.txt — tells Google "yes, index everything".

3. Meta tags on every page:

  • <title> — shows on browser tab and Google results
  • <meta description> — short text in Google results
  • Open Graph tags — nice previews on WhatsApp/LinkedIn
  • Twitter Card tags — Twitter previews

4. Structured data (JSON-LD) — hidden JSON code telling Google extra info:

  • AccountingService — you're a CA firm in India
  • WebApplication — each calculator is a tool
  • BreadcrumbList — shows nav path in search results
  • FAQPage — makes FAQ accordion show in Google
  • BlogPosting — for blog articles

5. Internal linking — every calculator links to others. Every blog links to related blogs. Helps Google understand topic clusters.

6. Canonical URLs — each page declares its "official" URL.

7. Mobile responsive — Google ranks mobile-friendly sites higher.

8. Fast loading — static HTML + minimal JS = pages load in under 1 second.

9. HTTPS — all sites secure (automatic).

What you should do (off-page SEO)

Google also looks at other sites that link to yours (backlinks). Right now you have ~0. Some ways:

  • Quora — answer 5 tax questions per week. Link to your calculator.
  • Google Business Profile — claim your listing. Adds you to Google Maps.
  • Directories — Justdial, Sulekha, IndiaMART (free).
  • Reddit (r/IndiaTax, r/personalfinanceindia) — answer questions, link when helpful.
  • Guest blog — write for other finance blogs in exchange for a backlink.

A separate web application at app.taxmitrafinance.com. Built with Next.js (React framework). Hosted on Vercel. Uses Supabase for data.

Why a separate site?

  • Different purpose: Marketing is for public discovery. Portal is for paying clients.
  • Different technology: Portal needs login, database, secure file storage.
  • Different hosting: Marketing = free static (GitHub Pages). Portal = server-capable (Vercel).
  • Cleaner experience: Clients see a focused dashboard, not marketing pages.

How data is protected

Supabase has Row-Level Security (RLS). Rules ensure: clients only see their own data; bugs can't leak one client's data to another; admins see all (you're marked as super_admin). Enforced at the database level for maximum security.

File storage

When a client uploads a Form 16:

  1. File saved at documents/{client-id}/{random-uuid}.pdf in Supabase Storage
  2. Each client has their own private "folder"
  3. Files only downloadable via "signed URLs" that expire after 1 hour
  4. Even guessing another client's file path is rejected by Supabase

9. PWA

PWA = Progressive Web App. A normal website browsers can install like a native app.

What clients see

On Android (Chrome)

  1. Visit app.taxmitrafinance.com and log in.
  2. After a few seconds, our "📲 Install TaxMitra Portal" banner pops up.
  3. Tap Install. App icon lands on home screen.
  4. Tap icon → opens like a native app, full-screen, no browser UI.

On iOS (Safari)

  1. Visit app.taxmitrafinance.com in Safari.
  2. Tap Share button (square with arrow).
  3. Scroll down → tap "Add to Home Screen".
  4. Confirm. App icon on home screen.
  5. Tap → full-screen, no Safari chrome.

On desktop (Chrome / Edge)

  1. Visit app.taxmitrafinance.com.
  2. URL bar — right side has an install icon (monitor with down arrow).
  3. Click → "Install TaxMitra Portal" dialog → Install.
  4. App opens in its own window.

11. Client Experience

A typical client's journey

1. Discovery — They Google "income tax calculator FY 2025-26" → land on your calculator page. They use it. See "Book Free Consultation". Click → Contact form.

2. First meeting — You consult. Onboard them as a paying client.

3. Signup — Go to app.taxmitrafinance.com/signup → verify email → log in.

4. First filing — They see "Welcome" but no filings yet. You create their filing. Now they see "Documents Requested" status.

5. Upload documents — Click Documents → drag-drop Form 16, Form 26AS, etc. Each shows with category labels.

6. Track progress — Get emails as you advance status. Can also open app anytime → see timeline. 🔔 bell with unread count.

7. Filed — You upload ITR acknowledgement via admin. They get email "Filed Successfully" + "Your CA shared a document". Download from portal.

8. Refund tracking — You update to "Refund Pending" with expected amount. When refund hits bank, mark "Refund Credited". Done!

What they can do anytime

  • Update profile (PAN, GST number, address)
  • Reset password if forgotten
  • Download any document you've ever shared
  • See past filings (FY 2024-25, FY 2025-26, etc.)
  • Install the app to their phone (PWA)

13. Troubleshooting

"Site is down" / "Vercel deploy failed"

  1. Open Vercel → your project → Deployments.
  2. Find recent deploy with red ❌.
  3. Click → see build log → look for "Error:" lines.
  4. Common: forgot to commit a file, syntax error in code, missing env variable.
  5. Share the error with me — I'll patch it.

"Emails aren't being sent"

  1. Resend dashboard → Emails tab → see if attempts listed.
  2. If "Bounced" → recipient email invalid.
  3. If not listed → check Vercel env var RESEND_API_KEY is set.
  4. Check Vercel function logs for [email] errors.

"I can't log in to the portal"

  • Try Forgot Password → check inbox (and spam) for reset email.
  • If reset email doesn't come, check Supabase Dashboard → Authentication → Users → confirm account exists.
  • If your account doesn't exist, sign up fresh.

"I made changes but the site doesn't show them"

  • GitHub Desktop — did you click both Commit to main AND Push origin?
  • Hard-refresh your browser (Ctrl+Shift+R) to bypass cache.
  • Wait 2 minutes — GitHub Pages can take time.

"A calculator is showing wrong numbers"

  • Math is in assets/calc.js.
  • Send me the specific inputs + expected vs actual output.

"Sitemap not updated for new blog posts"

  • Check GitHub Action for "Build blog from Markdown". Should run automatically.
  • Vercel → Actions → latest run → check logs.
